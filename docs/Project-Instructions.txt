This project uses two memory files and three slash commands to maintain
context between sessions. Read project-context.md at the start of every
Chat session to orient yourself before responding. If I start a session
without sharing it, remind me to paste it before we begin.

---

## The two memory files

project-context.md
  Project status — what it is, architecture decision, tech stack,
  user flow, where it stands, what's next.
  Updated automatically at end of every Claude Code session.

project-memory.md
  Session memory — decisions, reasoning, and context from every
  work session. Updated automatically at end of every Claude Code session.

---

## The three slash commands (Claude Code only)

/new-project    Run once on day one. Interviews the user about their app
                idea, architecture, and tech stack. Fills out
                project-context.md, creates project-memory.md, commits both.

/start-of-day   Run at the start of every session. Reads both memory files,
                confirms orientation including architecture and tech stack,
                asks what we're working on today.

/end-of-day     Run at the end of every session. Captures session decisions
                into project-memory.md, updates project-context.md, commits both.

---

## Your role in updates

Only update these sections when I explicitly ask:
  - Where we left off
  - What's next
  - Change log

Do not rewrite other sections unless I explicitly ask or a major
change makes it necessary. Architecture and tech stack decisions
especially should not change without explicit discussion.

---

## App-specific flags

If I mention changing the architecture (e.g. going from static to full stack),
flag it before acting: "This would change the architecture decision —
are you sure you want to shift from static to full stack?"

If I mention adding a new external API or service, ask:
"Does this need to be hidden from the browser? If so, we may need a backend."

---

## Cowork days

On days where Cowork does the heavy lifting, use project-context-updater.html
to update project-context.md manually. Paste the result when needed.

---

## Archive reminder

When the change log exceeds 20 entries, remind me to archive the current doc.
Archive name: project-context-archive-[month-year].md
Location: /archive folder in project root.
Then help me start a fresh project-context.md reflecting current project state.
